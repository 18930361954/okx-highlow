# 回测真实性三层验证

## 验证方法论

**问题**:如何证明回测数字不是"用未来数据选参数"的过拟合幻象?

**答案**:三层交叉验证。任一层不过关的策略都不上线。

| 层 | 方法 | 判定 |
|---|---|---|
| 1. Walk-Forward | 数据切 18 月训练集 + 6 月测试集,训练集选参数,测试集 out-of-sample 跑,看还赚不赚 | 测试集正收益 + MDD 不显著恶化 |
| 2. 滑点敏感性 | 同参数在 0/10/30/50bp 滑点下跑,看 pnl 曲线 | 10bp 滑点下仍稳定盈利 (OKX 主流币日常滑点区间) |
| 3. 仓位封顶 | 单笔 margin 硬顶 1500 USDT,避免复利数学放大到 OKX 流动性无法承接 | 数字规模现实可行 |

## 回测口径 (悲观)

- 初始 140 USDT · 每笔固定 10% margin · 单笔封顶 1500 USDT
- 杠杆:BTC/ETH 100x,SOL 50x (实盘 API 上限)
- 手续费:taker 5bp × 开+平 = 10bp of notional
- 滑点:10bp (OKX 主流币日常区间)
- Funding:3bp / 8h (中性市场估算)
- 桶回测:前桶阴阳 → 下桶浮动挂单 → 桶内逐 K 判 TP/SL,SL 优先保守
- 数据:2 年 730 天 (2024-07 → 2026-07)

**观察**:
- 4H 组合最脆弱,滑点 > 20bp 就爆仓
- 6H/12H 组合能承受 30bp 滑点
- 50bp 极端场景下无策略幸存 (数学上,tp=0.6-1.5% 扣完手续费+滑点后期望值变负)

**OKX 现实滑点参考** (来源:多次实盘观察):
- BTC/ETH 主流币,单笔 < 5000 USDT: 日常 3-8bp
- 快速行情(5 分钟内价格波动 > 1%): 15-30bp
- 闪崩瞬间(30 秒内 -5%): 50-100bp
- 单笔 > 10000 USDT: 加 5-15bp 深度成本

**结论**:采用 **10bp 通过 = 稳健**的标准。10 个策略全部通过。

## 层 3: 仓位封顶验证

不封顶时 ETH 6H 2 年 140 → 12.5 亿 USDT (89587328%),数学幻觉。
封顶 1500 USDT/笔后:

| 策略 | 无封顶 2 年 | 1500 封顶 2 年 | 现实可期 |
|---|---|---|---|
| ETH 6H | +89587328% | +77074% | 是 |
| SOL 6H | +3864592% | +38000% | 是 |
| BTC 6H | +267061% | +12000% | 是 |

**结论**:采用 1500 USDT 单笔封顶。到 15000 USDT 余额后 margin 不再增长,数字回到现实规模。

## 上线前必看

1. **半年 Test 集收益 = 未来半年收益期望**:6 月 test 段收益 +65% ~ +444%,对应月化 +9% ~ +34%
2. **滑点是实盘生死线**:若持仓时被闪崩打到 50bp 滑点持续几笔,策略可能连续亏损
3. **建议实盘启动前**:先跑模拟盘 3-7 天,记录每笔实际滑点分布,若均值 > 15bp 立即降杠杆或换策略
4. **1D 已剔除**:虽然 walk-forward 显示 1D 也能赚,但 test 段风险指标恶化,不采用
