# 6 账户策略总览 (每账户独立, 不串联)

## 架构

- **6 个账户完全独立**:各自 OKX API 三件套、独立余额/连亏/熔断状态、独立 scheduler cron
- **共享一份 data/trades.db**:trades / state 表加 `account` 列区分,方便汇总日报
- **每账户 = 一个 signal_bar + 三币独立参数**:BTC/ETH/SOL 共用信号周期,但 float/tp/sl 各自最优
- **仓位控制**:每笔 = 余额 × 10% 保证金,三币同时满仓时总保证金 = 30% 余额
- **杠杆**:BTC/ETH 100x,SOL 50x(实盘 API 上限)

## 关键说明

**"不串联" = 6 个账户彼此隔离**:
- 每账户读 OKX 的 API key 是自己的,只看到自己的持仓/挂单
- 每账户的余额状态存 db.state 里,主键 (account, key) 分开
- 每账户的成交/结算/连亏/熔断都不影响别的账户
- scheduler job id 带账户前缀,cron 触发时只调本账户的 signal/reconcile

**总仓位控制**:
- 每笔 = 余额 × 10% 保证金(顶层 `strategy.position_pct: 0.10`)
- 三币同时都持仓时:保证金总额 = 3 × 余额 × 10% = 30% 余额
- 100x 杠杆下名义仓位 = 保证金 × 100 = 3000% 余额,但每笔都是独立的 TP/SL 单
- 单笔 SL 命中 = 30 × leverage × SL% USDT(如 BTC 100x sl=1.5% 时亏 45U;SOL 50x sl=2% 时亏 30U)
- 熔断保护:3 笔连亏后暂停 24 小时(避开策略失效期)

**为什么共用 signal_bar**:
- scheduler 简单:一个账户一个 signal_bar → N 次 cron/日,三币共用
- 三币若各自不同 signal_bar,cron 会冲突且难对齐

**SOL 50x 特别说明**:
- 100x 是 OKX BTC/ETH 的常见上限;SOL 只支持 50x
- 我最初回测都用 100x → SOL 数字虚高。重跑 SOL 50x 后:参数最优点完全一致,收益/MDD 都约减半
- 现在 config.yaml 每账户 SOL 显式覆盖 `leverage: 50`

**上线建议**:
1. 先用**模拟盘-bot14559 (12H 最保守)** 跑 3-5 天,观察每天 2 次挂单的完整生命周期
2. 确认 db 里日报 / 成交明细正确后,再启用其他账户
3. 实盘账户资金分批打入,建议先 100-300 USDT/账户
