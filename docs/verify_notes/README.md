# verify_notes

存放"改动可能让实盘和回测拉开距离"这类变更的**上线前验证记录**。

## 什么改动需要写记录

判断标准：改动是否影响"实盘 vs 回测口径一致性"？

- ✅ K 线源 / 桶时区对齐（HK vs UTC）
- ✅ ctVal / lotSz / 张数精度
- ✅ 浮动 / TP / SL 计算公式
- ✅ 手续费 / 资金费 / 滑点估算改口径
- ✅ 信号周期切换或 pair_overrides 调整
- ❌ 纯 UI / 日志 / 面板展示优化
- ❌ 内部重构、类型改动、测试补充

拿不准 → 写。宁滥勿缺。

## 文件命名

`YYYY-MM-DD-<slug>.md`

例：`2026-07-17-6H-12H-utc.md`（跑 7 天后核对 6H/12H 桶对齐修复）
