## 一、策略规格（最终版）

### 1.1 核心参数

| 项 | 值 | 说明 |
|---|---|---|
| 标的 | BTC-USDT-SWAP、ETH-USDT-SWAP | 双品种并行运行 |
| 本金 | 75 USDT 起 | 子账户隔离 |
| 仓位 | 余额 × **10%** | 动态按当前余额计算 |
| 杠杆 | **100×** | 实盘需在 OKX 设置该交易对杠杆 |
| 浮动 | 信号日 high/low × **±0.15%** | high 上方 0.15% 挂空、low 下方 0.15% 挂多 |
| 止盈 TP | **1.2%**（相对入场价）| 服务端绑 |
| 止损 SL | **0.5%**（相对入场价）| 服务端绑 |
| 方向过滤 | 信号日**收阳只挂多、收阴只挂空、平盘跳过** | 顺势过滤 |
| 熔断 | 连亏 **3 笔** → 暂停 **24 小时** | 持久化到 SQLite |
| 切档 | 账户净值首次 ≥ **800,000 USDT** → 永久固定 **1000 USDT 保证金** | 一旦触发永久生效 |
| 信号生成时刻 | 每日 **00:00 UTC** | 用前一日 00:00~23:59 UTC 的 1H K 线 |
| 挂单有效期 | 当日 24h | 23:59 撤销未触发挂单 |

### 1.2 阴阳定义（关键！）

```
day_open  = 信号日第一根 1H K 的 open
day_close = 信号日最后一根 1H K 的 close

阳线 (close > open)  → 当日收涨 → 次日只挂多单（low 下方 0.15%）
阴线 (close < open)  → 当日收跌 → 次日只挂空单（high 上方 0.15%）
平盘 (close == open) → 跳过
```

**直觉**：跟前一日的"赢家方向"做动量延续。

### 1.3 回测验证结果（180 天 / 本地 1H CSV）

| 指标 | BTC | ETH |
|---|---|---|
| 总收益率 | +246.96% | +927.92% |
| 月化收益 | +43.84% | +159.07% |
| 胜率 | 56.41% | 64.71% |
| 最大回撤 | 16.94% | 18.54% |
| 总笔数 | 39 | 51 |
| 盈亏比 | 1.99 | 3.58 |

**30 天滚动窗口 22 个**：
- BTC：95.5% 正收益、中位 +19.7%、最差 −16.9%
- ETH：90.9% 正收益、中位 +38.4%、最差 −18.5%

**已知风险**：5 月后两品种均出现边际衰减（胜率掉到 33–43%）。实盘需每周复跑滚动窗口监控。

## 三、数据流（运行时）

```
每日 00:00 UTC（APScheduler 触发）
  ↓
core/scheduler.py → 调用 strategy.compute_signals()
  ↓
core/okx_client.py 拉前一日 24 根 1H K 线
  ↓
strategy/high_low.py
  ├─ 算 high / low / open / close
  ├─ 判方向（阳线/阴线/平盘）
  └─ 算挂单价（high×1.0015 / low×0.9985）+ TP/SL 价位
  ↓
core/account_state.py.can_trade()
  ├─ 在熔断期？→ skip
  ├─ 净值≥80w 切档？→ margin = 1000，否则 = balance × 0.10
  └─ 返回 OK + margin
  ↓
execution/order_manager.py.place_algo_orders()
  └─ OKX algo 算法单（触发限价 + attach TP/SL）
  ↓
execution/position_monitor.py 终端显示「今日待触发订单 + 现有持仓」
  ↓
WebSocket 收到成交回调
  ↓
core/account_state.py.on_filled(pnl)
  ├─ 更新 SQLite trades 表
  ├─ 更新余额
  ├─ 连亏计数 +1（亏）或归零（盈）
  └─ 连亏 ≥3 → 写 cooldown_until = now + 24h，撤当日剩余单
  ↓
日终 23:55 → scripts/daily_report.py 生成当日报告
日终 23:59 → 撤所有未触发挂单 → 进入下一日
```

## 四、关键技术决策

| 决策点 | 选择 | 理由 |
|---|---|---|
| 订单类型 | OKX **algo 算法单**（trigger 限价 + attach TP/SL） | 一次下单绑 TP/SL，程序断网/重启不影响成交 |
| TP/SL 价格 | **服务端绑**（不靠程序监控） | 不依赖本地程序在线 |
| 状态持久化 | **SQLite**（trades + state 两张表） | 重启后恢复连亏计数、熔断截止时间 |
| 调度器 | **APScheduler** `CronTrigger(hour=0, minute=0)` | 比自己写 sleep 循环稳，自带异常处理 |
| 行情拉取 | REST 拉日线（每日 1 次），WS 只订阅持仓变更 | 不需要 1m 实时数据，省资源 |
| 配置管理 | 单文件 `config.yaml` + `.env`（API key） | 不重蹈旧项目 500 行 yaml 的覆辙 |
| 日志 | 标准库 `logging` + `TimedRotatingFileHandler` | 不引第三方 |
| 部署 | `python main.py` + systemd / Windows 任务计划 | 不上 Docker，单机够用 |
| Python 版本 | 3.10+ | 用 `match` 和 PEP 604 类型注解 |

## 五、config.yaml 模板（全文）

```yaml
# config.yaml — 全部可调参数都在这里

strategy:
  pairs: [BTC-USDT-SWAP, ETH-USDT-SWAP]
  position_pct: 0.10                   # 仓位比例
  float_pct: 0.0015                    # 浮动 ±0.15%
  tp_pct: 0.012                        # 止盈 1.2%
  sl_pct: 0.005                        # 止损 0.5%
  leverage: 100
  trend_filter: true                   # 顺势过滤开关
  max_consecutive_losses: 3            # 连亏熔断阈值
  cooldown_hours: 24                   # 熔断暂停小时数
  fixed_mode_threshold: 800000         # 净值≥此值切固定模式
  fixed_mode_margin: 1000              # 切档后单笔保证金（USDT）
  signal_time_utc: "00:00"             # 信号生成时刻（UTC）

system:
  log_level: INFO
  log_keep_days: 30
  db_path: data/trades.db
  daily_report_time_utc: "23:55"

account:
  env: demo                            # demo / live
```

`.env.example`：
```
OKX_API_KEY=your_api_key_here
OKX_SECRET_KEY=your_secret_key_here
OKX_PASSPHRASE=your_passphrase_here
```

## 七、已经做过的事 / 教训记录

### 7.1 OKX API 注意点
- **`/market/history-candles` 单次最多 1440 根**（≈60 天 1H K 线）。回测要≥60 天必须分页或用本地 CSV
- **杠杆要预先在 OKX 设置**（contract-instrument 设置接口）
- **algo 单的 `instId` 必须带 `-SWAP` 后缀**
- **demo 环境的 base URL**：`https://www.okx.com`，但请求要带 `x-simulated-trading: 1` header

### 7.2 策略校准教训
- 原版 TP 1.8% + 双向挂单 180 天亏 73% —— **必须叠顺势过滤 + TP 缩短到 1.2%**
- 100× 杠杆 + 10% 仓位 + 顺势过滤的盈亏比稳定 ~2，胜率 56-65%
- 5 月后边际衰减，要每周复跑滚动窗口

### 7.3 旧项目要搬的回测脚本
直接从旧 okxfive 项目 `scripts/` 拷过来：
- `backtest_high_low_limit_order.py` → `scripts/backtest.py`
- `diagnose_high_low_strategy.py` → `scripts/diagnose.py`
- `rolling_window_test.py` → `scripts/rolling_window.py`

回测用的历史 K 线 CSV：
- 旧项目 `data/BTC_USDT_SWAP_1H_12m.csv`
- 旧项目 `data/ETH_USDT_SWAP_1H_12m.csv`

直接 copy 到新项目 `csv_data/` 即可。
