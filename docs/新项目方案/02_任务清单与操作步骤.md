```python
class HighLowStrategy:
    def __init__(self, config: dict, logger):
        ...

    def compute_signal(self, pair: str, candles_1h: list[dict]) -> dict | None:
        """
        Args:
            pair: BTC-USDT-SWAP
            candles_1h: 前一日的 24 根 1H K 线（按时间升序）

              'pair': 'BTC-USDT-SWAP',
```

#### T08 — core/account_state.py
核心类 `AccountState`，方法：

```python
class AccountState:
    def __init__(self, db, config, logger):
        ...
    def can_trade(self, now: datetime) -> tuple[bool, str]:
        """检查熔断 / 余额是否足够。返回 (允许, 原因)"""
    def compute_margin(self, balance: float) -> tuple[float, str]:
        """返回 (margin, mode)。mode = PCT 或 FIXED"""
    def on_trade_filled(self, pnl: float, exit_time: datetime):
        """成交后更新连亏 / 切档 / 熔断 / 余额"""
    def is_in_cooldown(self, now: datetime) -> bool: ...
    def get_balance(self) -> float: ...
```

- 全部状态从 SQLite `state` 表读写
- 熔断逻辑：连亏 ≥3 时写 `cooldown_until = now + 24h`，下次 can_trade 看时间
- 切档逻辑：余额首次 ≥80w 时写 `fixed_mode_locked = true`，永久生效
- ✅ 验收：单测 — 模拟连亏 3 笔触发熔断、24h 后恢复、净值过 80w 后永久 FIXED

## 五、应急预案

| 场景 | 处理方式 |
|---|---|
| 程序 crash | systemd / 任务计划自动重启，重启后从 SQLite 恢复状态 |
| OKX API 限流 | okx_client 加指数退避重试 |
| 网络断开 | algo 单已在服务端，TP/SL 自动执行；恢复后撤当日剩余挂单 |
| 余额低于最小保证金 | account_state.can_trade() 返回 False，自动跳过 |
| 想紧急停 | `Ctrl+C` 优雅退出 + 登录 OKX 网页手动撤所有算法单 |
| 触发熔断后想强制恢复 | `python scripts/reset_cooldown.py`（需自己写，2 行代码） |
